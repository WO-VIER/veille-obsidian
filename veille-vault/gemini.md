# Gemini

## En une phrase
`Gemini` est l'ecosysteme de modeles et produits IA de Google que je suis en contexte, distinct de la note specifique [[gemini-cli|Gemini CLI]].

## Pourquoi une note separee
Toutes les actualites `Gemini` ne concernent pas directement `Gemini CLI`.

Cette note sert donc a :

- garder les signaux generaux sur l'ecosysteme Gemini ;
- eviter de surcharger [[gemini-cli]] avec des annonces produit ou modele trop larges ;
- relier uniquement les signaux pertinents au suivi des outils developpeur.

## Mises a jour

### 2026-04-15 - Gemini 3.1 Flash TTS
- Source : `Google AI Blog`
- Signal principal : disponibilite de `Gemini 3.1 Flash TTS` dans les produits Google, avec un angle synthese vocale expressive.
- Ce que j'en retiens : Google continue d'etendre les capacites multimodales de l'ecosysteme Gemini au-dela du code et du terminal.
- Interet pour la veille : c'est un signal de contexte sur la trajectoire generale de Gemini, mais pas une preuve directe sur les capacites de [[gemini-cli|Gemini CLI]].

## Reflexions provisoires
- Les annonces autour de `Gemini` peuvent aider a comprendre l'environnement dans lequel evolue `Gemini CLI`.
- Il faut garder une separation nette entre une capacite de modele ou de produit Google et une fonctionnalite vraiment disponible dans l'outil CLI.
- Cette separation rend la veille plus propre : [[gemini-cli]] suit l'outil developpeur, tandis que cette note suit l'ecosysteme plus large.

## Limites
- une annonce produit Google ne suffit pas a conclure sur l'usage developpeur ;
- le lien avec `Gemini CLI` reste indirect tant qu'une release ou une documentation CLI ne le confirme pas ;
- cette note doit rester courte pour ne pas elargir artificiellement le sujet de presentation.
