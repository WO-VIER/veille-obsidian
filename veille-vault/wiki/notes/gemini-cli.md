# Gemini CLI

## En une phrase
`Gemini CLI` est présent dans ma veille comme un acteur important du paysage, mais mon corpus actuel reste plus mince et demande encore des sources complémentaires pour dépasser la simple trace de release.

## Positionnement visible dans la veille
- outil suivi via release officielle ;
- présent dans les comparaisons communautaires du paysage ;
- intéressant pour la veille car il apporte un troisième point de comparaison face à `Codex` et `Claude Code` ;
- outil à surveiller en continu plutôt qu'à évaluer trop vite avec le corpus actuel.

## Apport de la documentation officielle
- la documentation et le README officiels confirment un positionnement terminal-first et open source ;
- `Gemini CLI` met en avant des outils intégrés, le support `MCP`, le mode headless, le checkpointing et les fichiers `GEMINI.md` ;
- la documentation officielle donne aussi une lecture claire de l'authentification : login Google, clé API, ou `Vertex AI` ;
- la cadence `preview` / `stable` / `nightly` est explicite, ce qui est utile pour une veille continue.

## Réflexions provisoires
- `Gemini CLI` mérite clairement une place dans le périmètre de veille.
- Mon corpus actuel montre surtout sa présence dans le paysage, pas encore une compréhension fine de sa trajectoire.
- C'est probablement l'outil qui demande le plus vite une source active officielle complémentaire si je veux aller plus loin.
- La documentation officielle le rend déjà plus lisible : ouverture du code, auth différenciée, support `MCP`, `GEMINI.md`, mode headless et cadence de release claire.
- Les captures récentes montrent aussi une cadence de maintenance active, même si toutes les releases ne sont pas riches en contenu analytique.
- Les annonces plus larges autour de l'écosystème [[gemini|Gemini]] doivent rester séparées de cette note, sauf si elles concernent directement le CLI.

## Limites
- je reste encore avec peu de releases recoupées par rapport à `Codex` ;
- une partie de ce que je sais vient d'une source communautaire ;
- l'évaluation de l'outil reste donc provisoire et doit être enrichie par d'autres releases ou par la documentation officielle.
